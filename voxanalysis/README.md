# VOX Analysis

The analysis component is organised into **two independent
parts** plus the coaching layer that ties them together.

```text
vox-analysis/        PART 1 — measure & score singing (the metrics engine + web viewer)
youtube-downloader/  PART 2 — fetch an original recording to compare against
scripts/ + openclaw-data/   coaching layer (Candi's OpenClaw pipeline + runtime data)
docs/                current reference documentation
```

The two parts are deliberately separate: you can run the analysis without ever
touching the downloader, and the downloader knows nothing about scoring. They
meet only when you compare a take against an original.

---

## Part 1 — `vox-analysis/` (the VOX analysis metrics)

Everything that measures and scores a vocal. Nothing here reaches the internet.

| Piece | Path | What it is |
|---|---|---|
| Engine | `vox-analysis/engine/` | The measurement core. `analyse_song.py` runs Praat-grade acoustics (jitter/shimmer/HNR/CPPS/formants), the deterministic scoring rubric, calibration against the 50-track professional pack, and all diagnostic blocks. Also holds `calibration/`, `knowledge/` (prescription map) and `tools/`. |
| Viewer | `vox-analysis/viewer/` | The local web app (`http://127.0.0.1:8766`) — upload a take, see the pitch line against playback, the harmonic scope, and the calibrated report. Calls the engine by relative path. |

Every number the engine reports is measured or from a documented formula —
never generated by a language model. See `docs/metrics-methodology.md`.

### Run the engine directly

```bash
cd vox-analysis/engine
chmod +x install.sh && ./install.sh          # or: python3 -m venv voxai_env && pip install -r requirements.txt
sudo apt install ffmpeg                        # system dependency
python analyse_song.py "/path/to/take.mp3" --name "Aaron - Vienna" --separate-stems
```

### Run the web viewer

```bash
cd vox-analysis/viewer
pip install -r requirements.txt
python app.py        # open http://127.0.0.1:8766
```

See `vox-analysis/README.md` for the full tour.

## Part 2 — `youtube-downloader/` (fetch the original)

A small, self-contained tool that downloads an original/reference recording
from YouTube so a take can be compared against it. This is the **only** part of
the repo that goes out to the internet (via `yt-dlp` + `ffmpeg`).

Two front ends, one shared engine (`reference_dl.py`):

- **Web page** — `python app.py`, then open `http://127.0.0.1:8765` (paste a link
  or type a song, pick MP3/MP4, download).
- **Agent CLI** — `python3 youtube-downloader/fetch_reference.py "Adele Hello"`
  (prints a JSON manifest with the saved file path).

Downloads land in `openclaw-data/vox-coach/uploads/reference/` so the coaching
pipeline can pick them up. Keep `yt-dlp` current (`pip install -U yt-dlp`) —
YouTube breaks old versions frequently. See `youtube-downloader/README.md`.

> Copyright care: downloads are for private comparison/analysis only; delete
> reference media when it is no longer needed.

## The coaching layer

`scripts/candi_phase1.py` is Candi's OpenClaw intake/report pipeline. It uses
**both** parts — it runs the engine on a take, optionally calls the downloader
for the original, retrieves VOXAI knowledge, and writes coaching replies and
progress logs. Its runtime data (knowledge files, singer/song memory, uploads,
logs) lives under `openclaw-data/vox-coach/`, which stays put so existing local
data is never orphaned.

```bash
# knowledge sanity check
python3 scripts/verify_voxai_knowledge.py        # expect advanced_compliant: true

# prepare a take (optionally auto-fetch the original)
python3 scripts/candi_phase1.py prepare \
  --source-path "/absolute/path/to/recording.mp3" \
  --singer "Aaron" --song "Vienna" --artist "Billy Joel" \
  --fetch-reference
```

`--fetch-reference` makes the pipeline download the original automatically so
it is ready for comparison — no manual download step.

### Point Candi at a different engine location

```bash
export VOXAI_BACKEND_DIR=/absolute/path/to/vox-analysis/engine
```
